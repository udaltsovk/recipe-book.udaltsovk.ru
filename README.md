## Приложение "Книга рецептов"

Необходимо разработать SPA-приложение на Angular 21, которое будет содержать рецепты приготовления различных блюд и напитков.

### Требования  
- Данные по рецептам могут храниться как в json, который вы будете импортировать и парсить, так и в обычной const, как было на лекции. Минимальная модель рецепта (модель можно расширять по желанию, добавлять поля и менять названия модели/полей): - **1 балл** 
``` ts 
interface Receipt {  
    id: number;  
    name: string;  
    image: string;  
    description: string;  
    ...  
}  
```  
- Реализовать роутинг - как стандартный, так и с параметрами. По количеству маршрутов в целом понятно, что страниц в приложении должно быть минимум 3 (страницы с параметром считаем за одну) - **2 балла**
    - Должен быть как минимум 1 роут с параметром (`:param`)
    - И как минимум 2 стандартных роута (пустой роут тоже считается)
    - Добавить обработку несуществующих роутов - либо при помощи страницы 404 (предпочтительнее!), либо редиректом на главную страницу
- Разработать необходимое количество компонент для вашего приложения. Стремиться осознанно делить приложение на компоненты, чтобы они не содержали много html и кода. Повторяющуюся разметку также реализовывать в виде компонент, чтобы избегать дублирования логики/разметки  - **5 баллов**
    - Необходимо реализовать компоненты header, footer. Компоненты-врапперы страниц и далее по необходимости.
    - Как минимум два компонента должны "повторяться" в приложении. Это может быть как использование `@for`, так и просто использование одного и того же компонента в нескольких местах (например, кнопка)
    - Обязательно использовать control-flow в вашем приложении. Как минимум 1 раз вызвать `@if`; как минимум 1 раз вызвать `@for`. В случае необходимости (например, чтобы не обращаться к сигналу несколько раз за данными, использовать алиасинг `@if (mySignal(); as mySignal) {}`)
    - Хотя бы 1 раз использовать `input()`; хотя бы 1 раз использовать `output()`
    - Хотя бы 1 раз использовать property binding (`<tag [attribute]="value"></tag>`)
 - Реализовать хотя бы один pipe - **2 балла**
 - Реализовать хотя бы одну attribute directive - **2 балла**
 - На странице с рецептами должны быть фильтры - например, по времени готовки, типу рецепта (десерт, горячее, суп и тд), названию блюда, фильтр только для напитков и т.д  - **3 балла**
    - Необходимо реализовать хотя бы 2 фильтра при помощи template form и two-way data binding.
    - Фильтрацию можно проводить при помощи стандартных функций языка Typescript для работы с массивами.
 
 
### Дополнительное задание - 2 балла  
   Задеплоить получившееся приложение на github pages
 
 ### Дополнительные дополнительные задания - если нужно больше челленджа
 - Использовать в проекте какую-либо библиотеку компонентов, например, Taiga UI или Angular Material 
 - Реализовать форму для добавления нового рецепта - пользователь вводит название, описание, прикрепляет url изображения  
 - Реализовать редактирование рецепта - это можно сделать на основе формы выше
 - Сохранять и загружать данные по рецептам из LocalStorage - для этого хорошо бы реализовать *сервис*, с которыми вы познакомитесь на следующей лекции, но можно забежать вперед. То есть ваше приложение после перезагрузки страницы (F5) должно сохранять добавленные/измененные рецепты.
 - Добавить поддержку синтаксиса Markdown в рецепте - это не обязательно делать с нуля, существует библиотека.
 

### Ограничения  
- Не использовать `any` как тип в приложении - **минус 2 балла**
- Не создавать слишком большие компоненты (больше ~200 строк) - **минус 1 балл**
- Следовать принципам написания чистого и читаемого кода - переменные и функции должны называться по смыслу, код должно быть удобно читать. 
- Не вайбкодить. Использовать LLM для получения новых знаний/консультаций по документации или каким-то вопросам (также лучше будет использовать для этого преподавателя), но не для генерации готового кода.
